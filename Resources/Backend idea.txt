# AI-First HR Startup – Backend Architecture

## 1. Tech Stack

- Runtime: Node.js
- Framework: Express (faster than Nest for MVP)
- Language: TypeScript
- ORM: Prisma
- Database: PostgreSQL (Supabase Hosted)
- Auth: Supabase JWT verification
- AI: OpenAI API
- Queue: BullMQ (Redis via Upstash)
- File Parsing: pdf-parse
- Validation: Zod
- Calendar Integration: Google Calendar API
- Email: Resend / SendGrid

---

## 3. Core Modules

### Auth Middleware
- JWT validation
- Role enforcement
- Request context injection

---

### Candidate Module

Responsibilities:
- Resume upload
- Resume parsing
- Store extracted text
- Manage availability

---

### AI Module

Responsibilities:
- Resume analysis
- Skill extraction
- Match scoring
- Hire probability
- Feedback summarization

Async Flow:
1. HR clicks analyze
2. Job added to BullMQ
3. Worker processes resume
4. Result saved in AIProfile table

---

### Requirement Module

- Create tech requirements
- Define experience threshold
- Define match score threshold
- Auto-assign candidate on qualification

---

### Scheduling Module

Algorithm:
1. Fetch candidate availability
2. Fetch interviewer availability
3. Find overlapping slots
4. Use DB transaction
5. Create interview record
6. Mark slot as booked

DB Constraints:
- Unique (interviewerId + startTime)
- Unique (candidateId + startTime)

---

### Interview Module

- Create interview
- Store meet link
- Capture feedback
- Trigger AI feedback summarization

---

## 4. AI Prompt Engineering

Resume Analysis Prompt:

System:
"You are an AI hiring intelligence engine."

User:

- Required tech stack

Output JSON:
{

  seniority: string,
  extracted_skills: [],

  weaknesses: string,
